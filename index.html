How to read & use this

  1. MUST terms are ANDed (every candidate must have all of them). ANY terms are wrapped in (a OR b OR c). TITLES the same. Fewer MUST terms = more results but looser; more MUST = fewer but sharper.
  2. Regions add a location block like ("United Kingdom" OR Germany OR India…). X-ray reads the location printed on the profile, so toggling individual countries keeps results tight.
  3. Each platform just swaps the site: operator. LinkedIn for everyone; Xing is strong for Germany/EMEA; GitHub & StackOverflow surface the hands-on data/ETL people in Job 1.
  4. The auto-added -jobs -hiring -apply exclusions remove recruiter ads and job boards — that's the single biggest time-saver for accuracy.
  5. RUN COMBINED opens one tab across all selected platforms; the list below runs each platform separately when one engine over-mixes them.

Publish to GitHub Pages (2 min)

  1. Create a new repo on GitHub → upload this file as index.html.
  2. Repo Settings → Pages → Source = Deploy from a branch → branch main / folder /root → Save.
  3. Wait ~60s. Your tool is live at https://<your-username>.github.io/<repo-name>/ — fully static, nothing else to configure.
